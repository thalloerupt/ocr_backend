# 项目目标：
1. 实现一个pdf的实时翻译功能，其实现思路是通过rust项目获取pdf的每页段落布局bbox与文字信息，在pdf渲染页面添加对应的bbox覆盖层，覆盖层上为翻译后的内容，翻译通过接入大模型api来实现
# 项目架构：
1. 这是一个flutter + rust项目，通过flutter_rust_bridge来实现桥接
2. flutter部分，通过pdfrx库来实现pdf的显示，以及overlay覆盖层；接入大模型api功能也在flutter上实现
3. rust部分采用ocr_backend来实现pdf页面的ocr，获取段落bbox和text
# 项目设计：
1. 首页采用topbar+下面content的设计，topbar左边为文件打开按钮，点击可打开一个pdf，右边为（原文/翻译）按钮，窗口管理按钮，高度为48
2. content为pdf显示页面
3. 当pdf打开时应暂时禁止操作，等待预热结束，预热的工作是：运行rust部分代码，现将前5页的ocr结果识别出来，可以弹窗一个进度条显示百分比
4. flutter部分应创建一个页表，页的内容为每页的ocr信息，ocr信息应该为一个数组，数组存储一个block结构，其属性包括段落的bbox，text，以及其他必要信息；页表的大小预先设计为10,即可存放10页的ocr信息，当用户滚动到某一页时，若存在，应继续预加载下一页的ocr信息，若页表满，采用LRU算法进行淘汰插入；ocr应当是无阻塞的执行的，即不影响用户的滚动
5. 翻译内容采用流式输出
6. 当用户切换到翻译状态时，应开始翻译当前页面，将一个翻译完成的block内容缓存到block中，并标记完成翻译，当再次回到这一页时则无须重复翻译，除非当前页已淘汰；当用户滚动到另一页时，应立即停止上一页的翻译，即克开始当前页面翻译，未翻译完成的block翻译缓存丢弃。
7. 注意重组优化，在pdf滚动缩放时应当避免过度重绘
